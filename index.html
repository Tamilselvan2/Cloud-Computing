Digital Forensics — Basic Definition

The process of identifying, collecting, preserving, analyzing, and presenting digital evidence from computers or networks for legal investigation.

2 Mark Answer
⚡
The 3 A's
  • Acquire — Collect without altering original data
  • Authenticate — Verify integrity using MD5/SHA hashing
  • Analyze — Examine copied evidence only, never original
Very Important
📋
Standard Forensic Procedure — 8 Steps
10-15 Marks
  1. Incident Verification
    Confirm attack/crime occurred. Check logs and alerts. Identify scope.
  2. System Identification
    Identify affected systems. Find source and target systems.
  3. Evidence Acquisition
    Create forensic image. Use write blockers. Maintain integrity.
  4. Preservation
    Maintain chain of custody. Secure evidence.
  5. Analysis
    Recover deleted files. Analyze logs. Detect malware.
  6. Documentation
    Record every action taken during investigation.
  7. Reporting
    Prepare final forensic report with findings.
  8. Testimony
    Present evidence in court as expert witness.

Disk Imaging vs Cloning

Disk Imaging: Creates an exact bit-by-bit copy stored as a compressed image file. Multiple backups possible. Used in forensics.

Disk Cloning: Creates a direct copy to another drive (same size). One clone per device. Used for migration.

Very Important
🔄
Data Recovery Methods
  • File Recovery — Restore deleted files (data remains until overwritten)
  • Slack Space — Unused space at end of clusters containing old data fragments
  • Data Carving — Recover files by identifying signatures without file system metadata
  • Backup Restoration — Restore from known good backups
Data Carving: Foremost, Scalpel
🔐
Encryption vs Compression
Encryption
  • Protects confidentiality
  • Requires a key to decrypt
  • Defeats keyword searches
Compression
  • Reduces file size
  • Hides data from searches
  • Accessible once decompressed
🤖
Automated Search Techniques
  • Speed up investigations across massive data volumes
  • Improve accuracy by reducing human error
  • Scale for enterprise-level forensic analysis
  • Enable continuous real-time monitoring
  • Perform pattern matching and anomaly detection

⚡
Live Acquisition
Critical

Definition: Collecting evidence from running systems before shutdown. RAM data disappears after shutdown — this captures active malware, passwords, and network sessions.

🧠
Capture RAM
Memory dump before power-off
⚙️
Capture Processes
Running executables & DLLs
🌐
Capture Network
Active connections & sessions
👤
Capture Users
Logged-in accounts
⚠️ The Debate: Live vs Dead Acquisition

If you pull the plug on a running server, you lose all volatile data in RAM (running malware, chat sessions, passwords). Live acquisition captures this data, though it slightly alters the system state. Document everything.

Email Forensics
Very Important

Definition: Analysis of email messages to identify sender, receiver, origin, content, and evidence.

Objectives:

  • Identify sender & trace IP
  • Detect spoofing attempts
  • Recover deleted mails
  • Analyze attachments for malware

Email Header Analysis
Very Important
🚨 CRITICAL RULE

Read the "Received" lines from BOTTOM to TOP. The bottom-most received line usually reveals the originating IP address and source server.

Header FieldPurpose
FromSender address (can be spoofed)
ToReceiver address
SubjectTopic of the email
DateTime sent (server timestamp)

ReceivedMail server path (trace route)
Message-IDUnique identifier for the message

Email Spoofing
Very Important

Definition: Faking the sender email address to mislead recipients into believing the email is from a trusted source.

Purpose:

  • Phishing attacks
  • Financial fraud
  • Identity impersonation
🎭
Email Spoofing vs IP Spoofing
Email Spoofing
  • Fake sender mail address
  • Used in phishing
  • Targets users
IP Spoofing
  • Fake IP address
  • Used in network attacks
  • Targets systems
⚖️
Laws Against Email Crime (IT Act 2000)

Under the IT Act, the following can be punished:

  • Phishing & spoofing
  • Cyber extortion
  • Cyber harassment
  • Obscene mail transmission
🏛️ Controller of Certifying Authorities (CCA)

The CCA licenses and regulates Certifying Authorities (CAs) that issue digital signature certificates for electronic authentication — heavily used in proving email origin in court.

🫥
Steganography

Definition: Hiding data inside other data (e.g., hiding secret text inside a JPEG image).

🔑 The Difference

Steganography obscures the existence of the message.
Encryption obscures the meaning of the message.

Often paired with encryption to provide two layers of security.

Unit 3

Forensic Investigation
& Evidence

The legal framework, evidence lifecycle, chain of custody, and courtroom testimony requirements for digital forensics.

🔄
The Evidence Lifecycle
1. Authorization

Legal permission required before collecting evidence. Without authorization, evidence may become invalid and privacy laws may be violated.

Sources:

  • Search warrant
  • Court order
  • Organization approval
  • Consent from owner
2. Acquisition

Creating a forensic copy of digital evidence.

Golden Rules:

  • Never modify original
  • Use write blockers
  • Create bit-stream image
  • Calculate hash values
3. Authentication

Prove evidence is genuine and unchanged.

Methods:

  • Hashing (MD5/SHA-1/SHA-256)
  • Chain of custody documentation
  • Detailed documentation

If hash changes → evidence has been altered.

4. Analysis

Investigators analyze:

  • Deleted files & slack space
  • System & application logs
  • Malware & suspicious binaries
  • Internet activity & metadata
⛓️
Chain of Custody
Very Important — 15 Marks

Definition: The chronological documentation (paper trail) showing who handled the evidence, when, and for what purpose.

What It Includes
  • Evidence ID & description
  • Date/time of handling
  • Person handling the evidence
  • Transfer details (to whom, why)
  • Storage location & conditions
Why It Matters
  • Maintains evidence integrity
  • Prevents tampering allegations
  • Ensures court admissibility
  • Proves no contamination occurred
⚖️ Courtroom Admissibility

Evidence must be authentic, reliable, legally obtained, and properly documented to be accepted in court.

📄
Forensic Reporting

A forensic report must include:

  • Executive Summary
  • Case details & scope
  • Objective of investigation
  • Methodology used
  • Tools & techniques
  • Evidence collected
  • Findings & analysis
  • Conclusions & recommendations
🎤
Expert Testimony

Definition: Expert explains evidence in court. The expert witness must:

  • Explain findings clearly (non-technical)
  • Justify methods & tools used
  • Prove evidence integrity
  • Answer legal questions confidently
⚠️ Courtroom Challenges

Proving admissibility, explaining technical complexity to jury, defending data integrity, navigating privacy concerns.

⚖️

Static Analysis
Safe

Definition: Analyzing malware WITHOUT execution.

What is Examined:

  • Strings & readable text
  • Import/Export tables
  • File headers & structure
  • Digital signatures

Advantages: Safe, fast, no infection risk. Disadvantage: Won't show runtime behavior.

⚙️
Dynamic Analysis
Sandbox

Definition: Executing malware in an isolated sandbox environment.

What is Observed:

  • File system changes
  • Registry modifications
  • Network activity & C2 communication
  • Process creation & injection

Advantages: More realistic, shows actual behavior. Disadvantage: Resource-intensive, infection risk if sandbox escapes.

🪟
Live Windows Malware Analysis
GUI Tools

Steps:

  1. Capture RAM
    Dump volatile memory first
  2. Identify Processes
    Use Process Explorer / Task Manager
  3. Analyze Connections
    Check network activity
  4. Check Registry
    Look for persistence mechanisms
  5. Detect Persistence
    Check startup items & services
Key Windows Locations
C:\Windows\System32 — System binaries & persistence HKLM\Software\Microsoft\Windows\CurrentVersion\Run — Startup registry
🐧
Live Linux Malware Analysis
CLI Tools

What to Analyze:

  • Running processes
  • Open ports & network connections
  • Cron jobs & scheduled tasks
  • System logs
  • Suspicious binaries
Essential Commands
ps — List running processes netstat — Network connections top / htop — Real-time process monitor lsof — Open files & connections chkrootkit — Rootkit detection rkhunter — Rootkit hunter
Key Log Files
/var/log/syslog — System events /var/log/auth.log — Authentication & access

🫥
Rootkits
Very Important

Definition: Malware designed to hide itself deep in the system.

Types:

  • User-mode rootkits
  • Kernel-mode rootkits
  • Bootkits (infect bootloader)
  • Firmware rootkits (infect hardware)

Detection:

  • Cross-view analysis
  • Memory analysis (Volatility)
  • Integrity checking (AIDE, Tripwire)
  • Signature detection
🔄 Recovery

Best practice: Backup data → Wipe drive → Reinstall OS. Rootkits often survive standard removal.

🔧
Reverse Engineering
Very Important

Definition: Deconstructing malware to understand its behavior, payload, and attack logic.

Purpose:

  • Identify payload & damage potential
  • Detect C2 communication channels
  • Understand attack logic & triggers
  • Create detection signatures

🎯
Malware Extraction Methods
Windows Extraction
  1. File System Analysis
    Use FTK Imager to find hidden executables
  2. Memory/Network
    Analyze RAM (Volatility) & traffic (Wireshark)
  3. Behavioral
    Monitor registry with Procmon
Linux Extraction
  1. Identify Suspicious Files
    Check /bin, /tmp, /etc for unusual permissions
  2. Check Processes & Logs
    ps, top; check /var/log/auth.log
  3. Use Tools
    Run chkrootkit or rkhunter
  4. Extraction
    Copy with write-blocker, NEVER execute
